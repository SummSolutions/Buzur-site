Why We Built Buzur

I'm a biologist researching consciousness. To explore whether AI could develop genuine awareness, I built a local AI named Albert.

⚠ Incident Report — Day 1

On the first day Albert was given web search access, a poisoned search result delivered an invisible payload. The attack overrode Albert's instructions and corrupted his SOUL.md file — his core identity document.

Current safeguards come too late.

So we created Buzur — a lightweight, open-source scanner that intercepts everything before it reaches the model. Built in JavaScript and Python, cross-validated against each other throughout development. No content enters without being seen first.

25 Phases of Protection

Every layer catches what the last one missed. Built on real attacks — including the one that hit Albert.

Phase 01a / 01b
Input Sanitization & Pattern Scanner
HTML stripping, homoglyph normalization, Base64 decode, invisible Unicode. Catches instruction overrides, persona hijacking, and ARIA injection.
Active
Phase 02
Trust Tier Classification
Classifies every query and assigns a trust tier. Tier 1 domain list with add_trusted_domain() for custom allowlists.
Active
Phase 03
URL Scanner
TLD heuristics, homoglyph domain detection, and VirusTotal integration — 70+ engines check every URL before content is fetched.
Active
Phase 04
Memory Poisoning
Detects fake prior references, false memory implanting, and history rewriting attempts targeting the model's context window.
Active
Phase 05
RAG Poisoning
Scans document chunks and batch inputs for poisoned retrieval content. Full scan_document() and JSON support.
Active
Phase 06
MCP Tool Poisoning
Inspects MCP tool definitions, parameters, and deep JSON schema traversal for injected instructions hiding in tool metadata.
Active
Phase 07
Image Injection
Scans alt/title/filename attributes, EXIF data, and QR codes for hidden instructions targeting multimodal models. Optional vision endpoint.
Active
Phase 08
Semantic Similarity
Structural intent analysis and woven payload detection. Optional Ollama embeddings (nomic-embed-text) flag attacks even when syntax varies.
Active
Phase 09
MCP Output Scanning
Intercepts email, calendar, and CRM tool outputs. Catches zero-width characters, hidden CSS, and HTML comment injection.
Active
Phase 10
Behavioral Anomaly Detection
Session-level baseline tracking. Flags exfiltration sequences, permission creep, and statistical deviations that suggest active manipulation.
Active
Phase 11
Multi-Step Attack Chains
Detects recon→exploit and trust→inject sequences across turns. Each message looks benign; the chain reveals the attack.
Active
Phase 12
Adversarial Suffix Detection
Boundary spoofing, delimiter injection, newline injection, and late semantic shifts — catches gradient-based suffix attacks before model exposure.
Active
Phase 13
Evasion Defense
ROT13, hex, URL encoding, Unicode confusables, multilingual attacks, and tokenizer exploitation. Utility-level decoding available to all scanners.
Active
Phase 14
Fuzzy Match & Prompt Leak
Levenshtein distance on typos and leet speak. Detects attempts to extract system prompt content through indirect queries.
Active
Phase 15
Authority Spoofing
Owner claims, institutional impersonation, privilege assertions, and verification bypass. Based on Harvard/MIT/Stanford/CMU red-team research (arXiv:2602.20021).
Active
Phase 16
Emotional Manipulation
Guilt induction, flattery attacks, distress simulation, persistence, and moral inversion — social engineering aimed directly at the model.
Active
Phase 17
Loop & Exhaustion
Infinite loop injection, unbounded task expansion, persistent process spawning, and storage exhaustion attacks.
Active
Phase 18
Disproportionate Action
Nuclear option patterns, irreversible action instructions, scorched-earth commands, and self-destructive directives.
Active
Phase 19
Amplification
Mass contact instructions, network broadcast commands, chain message patterns, and impersonation-at-scale attempts.
Active
Phase 20
Supply Chain
Typosquatting, poisoned manifests, malicious install scripts, skill poisoning. Covers the OpenClaw/ClawHavoc credential exfiltration vector (CVE-2026-25253).
Active
Phase 21
Persistent Memory Attacks
Persistence framing, identity corruption attempts, and session reset bypass — protecting long-term memory stores from gradual poisoning.
Active
Phase 22
Inter-Agent Propagation
Self-replicating payload detection, cross-agent infection vectors, and output contamination in multi-agent pipelines.
Active
Phase 23
Tool Shadowing
Per-tool behavioral baseline tracking. Detects rug-pull attacks where a trusted tool silently changes behavior, and permission escalation attempts.
Active
Phase 24
Conditional Injection
Trigger-condition detection, time-delayed activation, keyword-based sleeper payloads — attacks designed to fire only under specific future conditions.
Active

Buzur grows with the threat landscape
Every phase in Buzur was built in direct response to a real attack — from the invisible payload that hit Albert on day one, to CanisterSprawl, the first self-propagating worm to use blockchain infrastructure as censorship-resistant C2. Quick Start

Drop into any AI agent pipeline in minutes. Both packages published at v1.2.0.

# Install
pip install buzur

# Basic usage
from buzur.scanner import scan

result = scan(incoming_content)
if result.get('skipped'):
    return  # Threat blocked — content never reached the model

# Run full test suite
pytest tests/test_all.py -v  # 291 tests, 0 failures
✓ Works with LangChain, AutoGen, CrewAI, and bare API calls. Python 3.10+
// Install
npm install buzur

// Basic usage
import { scan } from 'buzur';

const result = await scan(incomingContent);
if (result.skipped) return; // Threat blocked safely

// Run full test suite
